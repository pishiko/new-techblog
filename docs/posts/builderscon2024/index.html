builderscon 2024楽しかった〜レポ

2024-08-12

ビルダーズコン。2024年8月10日、埼玉会館で開催された技術好きのためのトークイベントです。
新型コロナウイルスの影響により実に5年ぶりの開催で、僕としても前回2019年の開催以来、2回目の参加でした。


2019年


当時まだ学生だった僕は、エンジニアが登壇するようなイベントには参加したことがありませんでしたが、当時激ハマりしていたゲームOxygen Not Includedの開発者が講演をするぞ、という情報を聞いて、実に不純な動機でbuilderscon 2019に参加しました。普段使っているプロダクトの裏側の話や、コンパイラーの話、キーボードの話もあれば、なんとスーパーカミオカンデの裏側の話まで、大変楽しいイベントだったのを覚えています。なんて素敵なイベントなんだろう、来年も是非参加しよう！と思っていたところ、残念なことに、最初で最後のbuildersconとなってしまいました。

そんなbuildersconが帰ってきたぞ！というわけで、今回も大変楽しいイベントでした。


講演


感想は以下になります。

  • 法律に準拠した本人確認システムを0から作った話 〜家計簿プリカB/43でのeKYC開発実例〜
    • 電子取引を行うアプリをリリースするにあたって、どのような法律があるのかを確認し、そのうえで必要な本人確認を自前で実装・運用した、という話。
    • システムとしては結構ゆるゆるで、重要なところは結局人なんだなーという感想でした。
    • エンジニア主体のスタートアップにおいて、法律とどう向き合えばよいのかという質問に対する「勉強しましょう」が、やっぱそうなのかあーという感じ。
  • React Server Component の疑問を解き明かす
    • テンプレートエンジンやSPA、SSRについて丁寧におさらいしたうえで、React Server Componentが新たに何をもたらすか、そして、RSCは旧技術への回帰、すなわち振り子ではなく螺旋である！という話でした。
    • 大変わかりやすかったです。触ってみたくなった。
    • 気を付けないと、サーバーで取得しているデータがフロントに漏れ得るという点がどうなんだろうと思いましたが、そもそもそれはSSRの頃から問題だったし、むしろ改善されているんだ、という風に受け取りました。この辺は実際に触ってみないとわからないなーと思いました。
  • すこやかなサービス運営のための PWG (Performance Working Group)
    • ダッシュボードを改善してチームを巻き込んだメトリクス確認会をやっていこうという話でした。
    • 学びしかない。そのまま持って帰ります。
  • ホー○ページビルダー 2024
    • インターネットにhtmlファイルを置くのって楽しいぜ！という話でした。
    • わかる。自分ももっと気軽に楽しくファイルを残していきたいなと思いました。
    • そして、もうちょっとだけ世に出ようとも思いました。
  • CLIジェネレータ「cyamli」によるコンソールアプリのスキーマ駆動開発
    • CLIのスキーマをyamlで書いたら雛形を生成してくれるくんを作ったぜという話でした。
    • 人の役に立つものを楽しんで作っている姿が素敵でした。
    • 久しぶりに日曜プログラミングやりて〜となりました。
  • 1000 行で実現する Linux on Browser
    • RISC-Vのエミュレーターを必要なモジュールだけRustで実装して、wasmでブラウザ上でLinuxを動かすために何が必要か、の話。
    • 以前bokuwebさんのNESエミュレーター実装を見て、自分もgolangでNESエミュを実装したことがあったのでとても楽しみにしていました。
    • NESに比べると、実装しなければならない命令セットの数も仕様もシンプルなんですが、エミュレートできるものが大変インパクトがあるという点で、CPUの動作原理を学べる自由研究として、すばらしい教材だなと思いました。真似しやすく楽しく学べるという意味で、初心者向けのタグも嘘ではないかもしれない。


どの講演も、知的好奇心を満たしてくれる素晴らしいお話でした。久しぶりに、「休みの日に無駄なコード書きて〜」となりました。
